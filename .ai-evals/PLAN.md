### 4. Dedup findings — break if spinning (fix efficiency)

**File:** `scripts/ai-implement.sh`, review loop

After iteration 2, compare current finding descriptions against previous iteration. If >70% overlap, mark as spinning and break.

```bash
if [[ $ITER -ge 3 ]]; then
  PREV=$(grep -hE '\[CRITICAL\]|\[HIGH\]' /tmp/review-$((ITER-1)).txt 2>/dev/null | \
    sed 's/.*— //' | sort -u || true)
  CURR=$(grep -hE '\[CRITICAL\]|\[HIGH\]' /tmp/review-${ITER}.txt 2>/dev/null | \
    sed 's/.*— //' | sort -u || true)
  if [[ -n "$PREV" && -n "$CURR" ]]; then
    OVERLAP=$(comm -12 <(echo "$PREV") <(echo "$CURR") | wc -l | tr -d ' ')
    TOTAL=$(echo "$CURR" | wc -l | tr -d ' ')
    if [[ "$TOTAL" -gt 0 && "$OVERLAP" -ge "$TOTAL" ]]; then
      issue_comment "## 🔄 Fix loop spinning — same findings as iteration $((ITER-1))
      Breaking review loop. Findings will be posted to PR for manual resolution."
      FINAL_VERDICT="PUSH_WITH_CAUTION"
      break
    fi
  fi
fi
```

## Additional fixes from post-analysis

### 12. `/ai` response pushes blocked by pre-push hook

**File:** `scripts/ai-respond.sh`

**Symptom:** Every `/ai fix` on a PR fails at `git push` because husky pre-push hook requires `.review-stamp` which `ai-respond.sh` never writes.

**Fix:** Add `git rev-parse HEAD > .review-stamp` after the commit, matching `ai-implement.sh` Phase 3 pattern.

**Rationale:** Kimi PR review runs after push anyway — the stamp satisfies the gate, the actual review still happens.
