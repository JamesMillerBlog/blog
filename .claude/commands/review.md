# Review Blog Post

Review and improve a blog post draft.

## Instructions

1. Identify the post to review:
   - If $ARGUMENTS provided, find that post
   - Otherwise, ask which post or review the most recently modified

2. Read the post content from `_posts/`

3. Review for:

### Content Quality
- Technical accuracy
- Clarity and readability
- Logical flow
- Completeness

### SEO
- Title length (50-60 chars ideal)
- Excerpt/meta description (150-160 chars)
- Heading structure (H2, H3)
- Keyword usage
- Internal links to related posts

### Style
- Consistent with James's voice
- Code examples are correct and well-formatted
- Images have alt text
- No broken links

### Grammar & Spelling
- Check for typos
- Consistent formatting
- Proper punctuation

4. Provide:
   - Summary of issues found
   - Specific suggestions with line references
   - Offer to implement fixes automatically

## Output Format

Present findings in categories:
- Critical (must fix)
- Recommended (should fix)
- Optional (nice to have)
